## What's Working

- **Monorepo**: Turborepo + pnpm workspace, 3 packages — `apps/api` (Hono, port 3001), `apps/web` (Next.js 15, port 3000), `packages/shared` (Zod schemas)
- **Database**: PostgreSQL via Docker, Drizzle ORM. Tables: `feeds` (with `authority_score` + `analyze_weight`), `articles`, `article_analysis`, `settings` (nullable `user_id` for multi-user forward-compat), `pipeline_runs`. View: `article_analysis_with_feed` (inner-joined for dashboard queries). Seed scripts for feeds + settings + the 2026-04-13 lab-proxy upgrades
- **Ingestion**: `rss-parser` + pure mapper + `feed-fetcher` service, `onConflictDoNothing` for URL dedup, inline title-similarity dedup via bigram Dice coefficient (48h window)
- **LLM pipeline** (fetch → analyze → summarize): central `llm-registry` with per-task prompts, unified `llmExecute()` with JSON extraction + model fallback, per-task model selection read from settings at call time. `analyze` produces relevance (0-100) + importance (0-100) + controlled-vocabulary tags in one call; `summarize` produces 2-3 sentence per-article summaries. `ALLOWED_TAGS` vocabulary templated into the analyze prompt via `{{ALLOWED_TAGS}}`
- **Analyze allocation** (Phase 10): pure `allocateSlots()` function distributes batch across feeds by weight with spillover + round-robin interleaving across feed buckets for cancel-fairness. 14-day recency window. Starvation-resistant: no single feed (e.g. arXiv) can dominate a run
- **Summarize policy** (Phase 11): value-first ordering `(relevance + importance) DESC, published_at DESC`, 14-day window, `feeds.enabled` filter. Low-value articles age out of the summarize queue rather than blocking high-value ones
- **Pipeline observability** (Phase 9): singleton `runPipelineWithProgress(trigger, onProgress?)` orchestrator shared by manual + scheduler paths. Every run inserts a `pipeline_runs` row. SSE stream at `GET /admin/pipeline/stream` emits per-article progress events. Cancellation via in-memory flag checked between LLM calls (in-flight call completes, partial results retained). History via `GET /admin/pipeline/runs`. Status snapshot at `GET /admin/pipeline/status` including next scheduled fire time. Cancel endpoint `POST /admin/pipeline/runs/:id/cancel`
- **Scheduler**: single `runSchedulerTick()` entry point calls `runPipelineWithProgress("scheduler")`. Cron expression read from settings (default `0 */2 * * *` = every 2 hours), hot-controllable via `scheduler_enabled` + per-phase `analyze_enabled` / `summarize_enabled` toggles + `analyze_batch_size` / `summarize_batch_size`. Cron parsed via `cron-parser` for next-run countdown
- **Composite ranking**: `GET /ranked` computes `(w_rel·rel + w_imp·imp + w_fresh·EXP(-λh) + w_auth·authorityScore + w_uniq·1.0) / Σweights` at query time (normalized 0-1). Weights, λ, and the freshness formula all read from settings per request — hot-tunable
- **Filter API** (Phase 13): `GET /ranked` accepts 13 query params (`q`, `sources`, `categories`, `tags`, `composite_gte`, `relevance_gte`, `importance_gte`, `published_at_gte`, `published_at_lte`, `sort`, `limit`, `offset`, `include_facets`). Zod-validated, 400 on malformed input. Wrapped response `{ rows, total, limit, offset, facets? }`. Opt-in facets via `?include_facets=1` with facet-minus-own-dimension semantics so chip counts don't zero out siblings. Tags use `arrayOverlaps` (ANY semantics); dates enforced as full ISO 8601 with offset; sort whitelisted across 6 fields
- **Settings**: DB-backed key/value store, CRUD API at `/settings`, auto-seed on startup, forward-compat multi-user via nullable `user_id`. Holds weights, λ, scheduler config, tag vocabulary, per-task LLM model selection (primary + fallback), UI theme
- **Web dashboard**: Next.js 15 App Router, Tailwind v4, shadcn primitives, warm-dark/amber "newsroom workstation" theme (Fraunces display + Geist Mono data + hairline borders + 0.25rem radius, no shadows/gradients, light/dark/system theme switcher). Pipeline control strip at top with idle / running / watching / history states, per-article progress ticker during runs, re-attach via `/status` polling when the user navigates back to the dashboard mid-run. Filter bar with 7 groups (search / sources / sort always open; categories / tags / thresholds / published behind a single "more filters" toggle), URL-as-source-of-truth via `useSearchParams` + `router.replace`, 300ms debounced search, facet counts inline on chips, in-page shimmer via `useTransition` + context during navigation, pagination footer
- **Web settings**: tabbed settings page at `/settings` with 7 sections (Scoring / Freshness / Scheduler / LLM Models / Tag Vocabulary / Theme / Feeds), per-tab dirty tracking + SaveBar, unsaved-changes Dialog, URL deep-linking via `?tab=`. `/feeds` redirects to `/settings?tab=feeds`
- **Shared**: Zod schemas for all DTOs — `Feed` (with `authorityScore` + `analyzeWeight`), `Article`, `ArticleAnalysis`, `AnalyzedArticle`, `CreateFeed` / `UpdateFeed`, `Setting` / `UpdateSetting`, `PipelineRun` + `PipelineStatus` + `PipelineProgressEvent` discriminated union, `rankedQuerySchema` + `rankedResponseSchema` + facet schemas, `parseRankedSort` helper, `DEFAULT_SETTINGS`, `ALLOWED_TAGS` (~39 entries)
- **Tooling**: Biome lint + format, Vitest **208 tests passing**, TypeScript strict mode, SwiftLint (iOS deferred), Turborepo caching, Docker PostgreSQL start/stop/reset
